# 勋章解锁反馈 · 验收测试用例

> 验证范围：连续达标勋章新增 + 记录后解锁弹窗 + 状态保留 + 支持关闭
> 环境：`frontend/` 目录下执行 `npm run dev` 启动本地服务

---

## 一、前置准备（必做）

1. 启动应用：
   ```bash
   cd frontend && npm run dev
   ```
2. 打开浏览器 DevTools → Application → IndexedDB → `carbon-footprint-db`，右键 **Delete database**，清空历史数据后刷新页面。

---

## 二、用例列表

### TC-01 初始状态（无记录）
**步骤**：刷新页面后观察
- [ ] 勋章区：所有 7 枚勋章均为灰色未解锁态
- [ ] 页面顶部：无任何勋章 Toast 提示
- [ ] Dashboard「本周排放目标进度」：4 个分类进度为 0，状态均为「进度良好」

---

### TC-02 基础徽章解锁（tree / bike / vegan）
**步骤 A · tree 种树徽章（累计 120 积分）**
1. 通过调整记录项累计积分≥120。推荐组合：
   - 连续 20 次记录：**出行 → walk（步行）/ bike（骑行） + 任意公里数**（每次12积分，约11次即可达132）
2. **观察**：
   - [ ] 最后一次提交后，页面顶部弹出 Toast，图标 🌳，名称「种树徽章」
   - [ ] Toast 含名称、解锁条件、关闭按钮 ✕
   - [ ] 底部进度条在 6 秒内从满到空，之后 Toast 自动关闭
   - [ ] 手动点击 ✕ 也可立即关闭
   - [ ] Toast 关闭/消失后，勋章区「种树徽章」**仍保持绿色解锁态**（状态保留）

**步骤 B · bike 骑行达人（低碳出行 8 次）**
1. 继续累计 8 条 bus/metro/bike/walk 类型的出行记录（若前面已够则无需重复）
2. **观察**：
   - [ ] 第 8 条提交后，Toast 🚴 弹出「骑行达人」
   - [ ] 「种树徽章」在勋章区仍保持解锁（未被重置）

**步骤 C · vegan 素食主义者（素食 10 次）**
1. 记录 10 次：**饮食 → vegan + 任意餐数**
2. **观察**：
   - [ ] 第 10 条 Toast 🥗「素食主义者」弹出
   - [ ] 已解锁的 tree/bike 在勋章区依旧解锁

---

### TC-03 周达标类徽章解锁（核心新增）

**目标**：造出「本周数据」使得所有分类排放≤周目标，并验证对应徽章。
> 默认周目标：出行 25kg / 饮食 20kg / 用电 35kg / 购物 15kg
> 记录日期为今天（本周一至周日内）

**推荐记录组合（总量刚好低于目标）**：
| 分类 | 类型 | 数量 | 本次排放 | 累计 |
|---|---|---|---|---|
| transport | bus | 100 km | 6.0 kg | ✅ ≤25 |
| diet | mixed | 5 餐 | 8.0 kg | ✅ ≤20 |
| electricity | kwh | 20 度 | 11.6 kg | ✅ ≤35 |
| shopping | cost | 100 元 | 4.0 kg | ✅ ≤15 |

**步骤**：
1. 按上表记录 4 条数据（日期均为今天）
2. **观察 Dashboard 本周进度**：
   - [ ] 4 个分类进度条均为绿色，百分比合理（<80%）
   - [ ] 建议栏无预警
3. **勋章解锁观察**：
   - [ ] 全部分类达标的那条记录（通常为最后一条）提交后，同时弹出 2 个 Toast：
     - ⭐「周达标新星」—— 任意分类连续达标 1 周
     - 🎯「全分类达标」—— 所有分类同一周达标
   - [ ] 2 个 Toast 可分别/一并关闭
   - [ ] 关闭后勋章区 5 枚（tree/bike/vegan + 2 枚新增）均保持绿色解锁
4. 刷新页面（F5）：
   - [ ] 顶部 Toast **不重复弹出**（仅在解锁当次触发）
   - [ ] 勋章区 5 枚勋章 **仍为解锁态**（IndexedDB 持久化）

---

### TC-04 预警功能验证（80%/95% 阈值）

**步骤 A · 接近目标（warning 80%~95%）**
1. 先清空 IndexedDB 重新开始
2. 记录一条出行数据：**car（自驾） 100 km** → 排放 21 kg（出行目标 25kg，进度 84%）
3. **观察**：
   - [ ] Dashboard「出行」进度条为 **琥珀黄色**，状态标签「接近目标」
   - [ ] 「减碳建议 & 预警」顶部出现琥珀色卡片：「出行分类接近目标」，含剩余额度 4.0 kg 和针对性建议

**步骤 B · 即将超标（danger ≥95%）**
1. 再记录一条：**car 18 km** → 累计 3.78 kg，出行总 24.78 kg（进度 99.1%）
2. **观察**：
   - [ ] 进度条变 **红色**，状态「即将超标」
   - [ ] 预警卡片变红色 ⚠️：「出行分类即将超标」，剩余额度≈0.22kg
   - [ ] 建议栏排序：danger 项排在 warning 前（若同时存在）

**步骤 C · 超出目标（≥100%）**
1. 再加一条：**car 1 km** → 出行总 24.99 kg（超 100%）
2. **观察**：
   - [ ] 进度条下额外显示：「已超出目标 X%」
   - [ ] 预警仍按 danger 样式展示

---

### TC-05 周目标可编辑 & 持久化

**步骤**：
1. 在 Dashboard 本周进度卡片中，点击出行分类的 `25 kg`（虚线下划线）
2. 输入框出现，改为 `10`，按 **Enter** 或 **点击外部失焦**
3. **观察**：
   - [ ] 出行目标立即更新为 10 kg，进度条按新目标重算（可能变为 danger）
   - [ ] 刷新页面（F5）后：目标仍为 10 kg（IndexedDB 持久化）
4. 再次编辑，按 **Esc**：
   - [ ] 未保存修改，值恢复原状
5. 输入 `0` 或非数字：
   - [ ] 提交后不保存，维持原有合法值

---

### TC-06 连续达标 4 周（进阶徽章）

> ⚠️ 需要模拟日期，推荐两种方式任选：
> **方式一（推荐）**：DevTools → 在 `useCarbonStore` 的 `addRecord` 里打断点，提交时把 `record.date` 改为对应周的日期
> **方式二**：直接调用 IndexedDB 写入带伪造日期的记录

**目标**：
- 构造 4 个连续的周，每周都有 4 分类达标数据
- 触发 `week_streak_4` 🏅「月度坚持者」和 `perfect_month` 👑「完美月度」

**验证**：
- [ ] 满足条件的当次记录提交后，Toast 🏅 和 👑 依次弹出（按 id 顺序错开 0.12s）
- [ ] 关闭后勋章区 7 枚全绿
- [ ] 刷新后全部保留，Toast 不重复出现
